========================================================================
             VANGUARD EDR - WINDOWS SETUP & TESTING INSTRUCTIONS
========================================================================

Follow these steps to set up and run the Vanguard EDR Agent and the 
Centralized Server on your Windows system.

------------------------------------------------------------------------
STEP 1: INSTALL MICROSOFT SYSMON (REQUIRED FOR LIVE MONITORING)
------------------------------------------------------------------------
Vanguard listens to Windows Event Logs generated by Sysmon. If Sysmon 
is not installed on the system, live monitoring will not function.

1. Download Sysmon from Microsoft Sysinternals:
   https://learn.microsoft.com/en-us/sysinternals/downloads/sysmon
2. Extract the downloaded ZIP file.
3. Open "Command Prompt" (cmd) as ADMINISTRATOR.
4. Navigate to the extracted folder and run the installation command:
   sysmon.exe -i -n
5. Keep the Command Prompt open.

------------------------------------------------------------------------
STEP 2: TRANSFER THE PROJECT CODE
------------------------------------------------------------------------
Copy the following two directories from your Linux environment to your 
Windows machine (via USB, shared drive, or Google Drive):
- "vanguard_agent" (The client application)
- "vanguard_server" (The centralized server)

------------------------------------------------------------------------
STEP 3: INSTALL PYTHON DEPENDENCIES ON WINDOWS
------------------------------------------------------------------------
Ensure Python (version 3.9 - 3.11 is recommended) is installed on your
Windows system and is added to the system Environment Variable Path.

1. Open "Command Prompt" (cmd) inside the copied "vanguard_agent" folder.
2. Install the client dependencies:
   pip install customtkinter pywin32 tensorflow numpy

3. Open a separate "Command Prompt" (cmd) inside "vanguard_server" folder.
4. Install the server dependencies:
   pip install flask

*(Note: If TensorFlow takes too long or fails to install, you can skip
it. The agent will automatically fall back to rule-based heuristics
and CVE signature checks).*

------------------------------------------------------------------------
STEP 4: RUNNING THE EDR CONSOLE
------------------------------------------------------------------------
A. Run the Centralized Server:
   1. Open cmd in the "vanguard_server" folder.
   2. Start the server:
      python server.py
   *(The server will start hosting on http://127.0.0.1:5000)*

B. Run the Vanguard Client Agent:
   1. Open cmd in the "vanguard_agent" folder.
   2. Start the client GUI:
      python main_gui.py
   *(The status bar at the bottom-left should show "● Sysmon Active" 
   and "● Sync Server Online" in green).*

------------------------------------------------------------------------
STEP 5: HOW TO TEST THREAT DETECTION & ACTIVE MITIGATION
------------------------------------------------------------------------
To verify the EDR agent captures real threats, kills malicious processes,
deletes threat binaries, and reports them:

1. Test Case A: Real-Time File Watchdog (Always-On Protection)
   - Go to the "Ransomware Scanner" tab in the GUI.
   - You will see the "ALWAYS-ON BACKGROUND PROTECTION" status card.
   - Select or type a folder path (e.g. your Documents or Desktop folder) 
     in the folder path entry and click "Apply Watchdog to Selected Folder".
   - Create a dummy blank file inside that folder, and rename it to 
     "test_file.cerber" or "invoice.locky".
   - **What happens:** The background watchdog thread immediately detects the 
     ransomware extension, deletes/quarantines the file from disk, logs a 
     "Ransomware Watchdog" alert inside the database, and displays an 
     active remediation warning window.

2. Test Case B: Suspicious Process Remediation (Process Terminate & Delete)
   - Open File Explorer and type "%TEMP%" in the address bar to go to 
     your user's temporary folder.
   - Copy "notepad.exe" from "C:\Windows\System32\" and paste it here.
   - Rename the copied file to "vanguard_test.exe".
   - Double-click "vanguard_test.exe" to run it.
   - **What happens:** The Sysmon listener captures the process start. 
     Vanguard immediately intervenes, terminates the process ID (killing 
     vanguard_test.exe), deletes "vanguard_test.exe" from your Temp folder,
     logs an "Incident Response" mitigation alert inside the SQLite database,
     and syncs the mitigation history to your Flask server console!

3. Test Case C: Sandbox XML Analyzer (Grading Demo)
   - In the Vanguard Client GUI, go to the "Threat Intelligence" tab.
   - Under the "Manual Analysis Sandbox" section, click "Browse XML".
   - Select the "api.xml" file in your folder.
   - Click "Analyze Sandbox Log".
   - This parses the XML, matches the dissertation sequence, triggers a 
     warning dialog, and maps it to T1486 (Ransomware) and CVE-2019-11510.

4. Test Case D: Centralized Web Console (Mitigation Sync)
   - Open your web browser on Windows and navigate to: http://127.0.0.1:5000
   - You will see the centralized dashboard displaying the consolidated 
     logs and the active remediation histories synchronized in real-time.
